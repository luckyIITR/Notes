DeepSeekMath: Pushing the Limits of Mathematical Reasoning

An open-source math reasoning model that matches closed-source giants — by combining a massive web-scraped math corpus, careful pretraining, and a new RL algorithm called GRPO that eliminates the critic network entirely.

DeepSeekMath: Pushing the Limits of Mathematical Reasoning in Open Language Models
Zhihong Shao, Peiyi Wang, Qihao Zhu, Runxin Xu, Junxiao Song, Mingchuan Zhang, Y.K. Li, Y. Wu, Daya Guo  ·  arXiv 2024  ·  DeepSeek-AI
1 · Overview & Key Contributions

🎯 What This Paper Does

  • Trains DeepSeekMath-7B — a 7B open-source model that achieves 51.7% on MATH, competitive with GPT-4 (42.5% without tools) and Gemini Ultra.
  • Introduces DeepSeekMath Corpus — 120B tokens of high-quality math content web-scraped from Common Crawl.
  • Proposes GRPO (Group Relative Policy Optimization) — a new RL algorithm that replaces the PPO critic network with group-normalized rewards, cutting memory and compute significantly.
  • Provides a unified view showing that SFT, RFT, DPO, PPO, and GRPO are all special cases of the same underlying optimization framework.
The central insight: mathematical reasoning ability comes primarily from pretraining data quality, not model scale. A well-pretrained 7B model on the right data outperforms much larger models trained on general corpora.
2 · Three-Stage Training Pipeline

🏗️ From Coder to Math Reasoner

Stage 1
Math Pretraining

Continue pretraining DeepSeek-Coder-Base-7B on 120B tokens from the DeepSeekMath Corpus. Builds deep mathematical knowledge from scratch.

Stage 2
Supervised Fine-Tuning

Fine-tune the pretrained model on 776K math instruction pairs — problems with step-by-step chain-of-thought solutions. Creates DeepSeekMath-Instruct.

Stage 3
GRPO Reinforcement Learning

Further align with RL using GRPO and outcome rewards. Model learns to self-improve by comparing multiple sampled solutions. Creates DeepSeekMath-RL.

  • Starting point: DeepSeek-Coder-Base-7B — chosen because code pretraining provides strong structured reasoning and formal logic foundations.
  • Each stage progressively improves mathematical reasoning with a different type of supervision signal.
3 · DeepSeekMath Corpus — The Data Engine

📦 120B Tokens of High-Quality Math Content

  • Web-scraped from Common Crawl using an iterative URL-based filtering pipeline.
  • Seed: OpenWebMath (a known high-quality math dataset). Use it to train a fastText classifier to identify math-relevant URLs.
  • Iteratively expand: recall math pages from Common Crawl → add quality URLs → retrain classifier → repeat for 3 rounds.
  • Deduplicated and filtered for quality. Includes ArXiv math papers, math forums (StackExchange, AoPS), textbooks, and educational sites.

4 · The Problem with PPO for LLM Reasoning

🧩 Why Standard PPO Is Expensive

  • Standard PPO for LLMs requires loading four models simultaneously: the policy πθ, the reference model πref, the reward model rψ, and the critic/value model Vφ.
  • The critic Vφ is typically the same size as the policy — it doubles memory requirements and adds a separate training loop.
  • For a 7B model, this means holding ~28B parameters in GPU memory just for training.
  • The critic must be trained simultaneously with the policy — adding instability and requiring careful hyperparameter tuning.
Key question the paper asks: Do we even need a value network? For reasoning tasks with verifiable outcomes (math, code), can we estimate the baseline more cheaply?
5 · GRPO — Group Relative Policy Optimization

⚡ Eliminate the Critic — Use the Group as Baseline

  • For each question q, sample a group of G outputs {o1, o2, …, oG} from the old policy πθ_old.
  • Score each output with the reward model: {r1, r2, …, rG}.
  • Compute the advantage of each output relative to the group — normalize by the group's mean and standard deviation. No critic needed.
  • Update the policy using a PPO-style clipped objective with these group-relative advantages.
GRPO Advantage Estimation (no critic)

ri — reward of the i-th output in the group
mean({r}) — mean reward across all G outputs for this question
std({r}) — standard deviation of rewards in the group
The group mean acts as the baseline — replacing the value network V(s)

7 · GRPO vs PPO

🔀 Key Differences

🔒 PPO (standard)
  • Needs a critic network Vφ (same size as policy)
  • Baseline = V(st) learned from scratch
  • Advantage = r + γV(s') − V(s) via GAE
  • 4 models in memory: policy, ref, RM, critic
  • Critic training adds instability
  • Works well for sequential state-action problems
  • Requires per-token value estimates
⚡ GRPO (DeepSeekMath)
  • No critic — baseline from group mean reward
  • Baseline = mean(r1…rG) for the same question
  • Advantage = z-score within the group
  • 3 models in memory: policy, ref, RM
  • Simpler, more stable training
  • Ideal for tasks with verifiable outcomes
  • Sequence-level reward (not per-token)

8 · Reward Design

🏆 What Counts as a Reward

  • DeepSeekMath uses outcome rewards — not process rewards. The reward is given only based on the final answer.
  • For math problems: r = 1 if the final answer matches the ground truth (exact match or symbolic equivalence), r = 0 otherwise.
  • No learned reward model (like InstructGPT) — the reward is a rule-based verifier that checks correctness programmatically.
  • This avoids reward hacking since the verifier cannot be fooled — either the math is right or it isn't.
Outcome Reward
r(o, q) = 1 if answer(o) == groundtruth(q) else 0
o — generated solution (chain-of-thought + final answer)
q — math question with known ground truth
Checked via symbolic math matching (handles equivalent forms like 1/2 = 0.5)
Rule-based verifiable rewards are a key feature of math/code RL — they are free from reward hacking and require no human labeling beyond the original problem-answer pairs.

10 · Full GRPO Training Algorithm

⚙️ Step-by-Step

  1. Initialize policy πθ from the SFT model. Freeze a copy as the reference model πref.
  2. For each training iteration, sample a batch of math questions {q1, …, qB} from the training set.
  3. For each question q, sample G outputs {o1, …, oG} from the current old policy πθ_old using temperature sampling.
  4. Score each output with the rule-based verifier: ri = 1 if correct, 0 if wrong.
  5. Compute group statistics: μ = mean(r1…rG), σ = std(r1…rG).
  6. Compute group-relative advantage for each output: Âi = (ri − μ) / σ.
  7. For each output token, compute the probability ratio ri(θ) = πθ(oi|q) / πθ_old(oi|q).
  8. Compute the clipped PPO-style objective plus the KL penalty against πref.
  9. Update πθ via gradient ascent. Set πθ_old ← πθ after each iteration.
  10. Repeat until policy converges or a fixed step budget is reached.

13 · Pros & Cons of GRPO

⚖️ Strengths & Limitations

Strengths
  • No critic — halves memory vs PPO
  • Simpler training loop
  • Rule-based reward — no reward hacking
  • Naturally exploits task's verifiable structure
  • Group sampling provides diverse learning signal
  • Generalizable to any verifiable task
Limitations
  • Requires verifiable reward (math, code only)
  • Outcome reward misses step-level errors
  • High variance if G is small
  • No improvement if all G outputs correct or all wrong
  • Group advantage collapses when σ ≈ 0
14 · Legacy & What Came Next

🚀 Impact on the Field

  • DeepSeek-R1 (Jan 2025) — DeepSeek's reasoning model trained purely with GRPO and outcome rewards. Achieves o1-level reasoning. Trained without SFT in the RL phase — the model learned chain-of-thought reasoning entirely from RL.
  • DeepSeek-R1-Zero — variant that applies GRPO directly to the base model (no SFT at all), showing RL alone can induce structured reasoning behavior.
  • Qwen-Math, Kimi-Math — subsequent models from other labs adopting the GRPO + verifiable reward paradigm for math.
  • DAPO (2025) — "Decoupled Clip and Dynamic sAmpling Policy Optimization" — extends GRPO with token-level advantages and dynamic group filtering to address the σ ≈ 0 collapse problem.
  • OpenAI o1/o3 — uses a similar process-supervised GRPO-like RL approach, though with additional process rewards on top of outcome rewards.
GRPO became the dominant RL algorithm for LLM reasoning — displacing PPO for math and code tasks due to its simplicity and efficiency. DeepSeek-R1's success validated that RL with verifiable rewards alone can unlock emergent chain-of-thought reasoning without human demonstrations.
