Policy Gradient Methods

A family of RL algorithms that directly optimize a parameterized policy using gradient ascent on expected return — without needing a value function as an intermediary.

1 · Motivation — Why Not Just Use Value Methods?

💡 The Case for Policy-Based Approaches

  • Value-based methods (Q-Learning, DQN) learn Q(s,a) and derive a policy indirectly via argmax. This requires a discrete, enumerable action space.
  • Policy gradient methods parameterize the policy directly as π(a|s; θ) and optimize θ directly.
  • They naturally handle continuous action spaces (e.g. robot joint torques) where argmax over actions is intractable.
  • They can learn stochastic policies — useful in partially observable or adversarial environments.
  • They are more compatible with function approximation — no need for the greedy argmax that causes instability in value methods.

2 · Policy Parameterization

📐 Representing π(a|s; θ) with a Function Approximator

  • Instead of a lookup table, the policy is a parameterized function — typically a neural network with weights θ.
  • The network takes state s as input and outputs a probability distribution over actions.
  • We optimize θ to maximize the objective J(θ) = E[Gt].
🎲 Softmax (Discrete Actions)
π(a|s;θ) = e^h(s,a,θ) / Σ e^h(s,b,θ)

h(s,a,θ) is the action preference (a linear or neural net function). Ensures valid probability distribution over all discrete actions.

📊 Gaussian (Continuous Actions)
π(a|s;θ) = N(μ(s,θ), σ²(s,θ))

Network outputs mean μ and variance σ². Actions are sampled from this Gaussian. Used in robotics, continuous control tasks.

3 · Objective Function

🎯 What We're Optimizing

  • The goal is to find θ that maximizes the expected return.
  • For episodic tasks, the objective is the expected total return from the start state.
Performance Objective
J(θ) = Eπθ[ G0 ] = Eπθ[ Σt=0∞ γt rt+1 ]
θ — policy parameters (neural network weights)
πθ — the parameterized policy
G0 — total discounted return from timestep 0
γ — discount factor
We want to perform gradient ascent on J(θ): θ ← θ + α ∇θ J(θ). The challenge is computing ∇θ J(θ) — the policy gradient theorem solves this.
4 · The Policy Gradient Theorem

📜 The Core Result

  • The Policy Gradient Theorem (Sutton et al., 1999) gives a tractable expression for ∇θ J(θ) that does not require differentiating through the environment dynamics.
Policy Gradient Theorem
∇θ J(θ) ∝ Σs μ(s) Σa Qπ(s,a) ∇θ π(a|s; θ)
μ(s) — on-policy state distribution (how often s is visited under πθ)
Qπ(s,a) — action-value function under current policy
∇θ π(a|s; θ) — gradient of the policy w.r.t. θ
Log-Derivative Trick (Likelihood Ratio)
∇θ J(θ) = Eπθ[ Qπ(s,a) · ∇θ log π(a|s; θ) ]
Uses identity: ∇ π = π · ∇ log π to convert to an expectation.
This expectation can be estimated by sampling trajectories — no need to know transition dynamics.
Key insight: ∇θ log π(a|s; θ) (the score function) is easy to compute via backprop through the policy network. The environment dynamics cancel out entirely.
5 · REINFORCE — Monte Carlo Policy Gradient

🎲 The Simplest Policy Gradient Algorithm

  • REINFORCE (Williams, 1992) estimates the gradient by substituting the actual sampled return Gt in place of Qπ(st, at).
  • It is a Monte Carlo method — a full episode must complete before any update.
  • Intuition: if an action led to high return, increase its probability. If low return, decrease it.
REINFORCE Gradient Estimate
∇θ J(θ) ≈ Σt=0T Gt · ∇θ log π(at|st; θ)
Gt — actual return from step t to end of episode
∇θ log π(at|st; θ) — score function (gradient of log policy)
Parameter Update
θ ← θ + α · Gt · ∇θ log π(at|st; θ)
6 · REINFORCE Algorithm

⚙️ Step-by-Step

  1. Initialize policy network parameters θ randomly.
  2. Generate a full episode by following π(·|·; θ): collect trajectory (s0, a0, r1, s1, a1, r2, …, sT).
  3. For each timestep t = 0, 1, …, T−1, compute the return: Gt = Σk=t+1T γk−t−1 rk.
  4. Compute the policy gradient estimate: ∇θ J ≈ Σt Gt · ∇θ log π(at|st; θ).
  5. Update parameters: θ ← θ + α · ∇θ J(θ).
  6. Repeat from step 2 until convergence.
7 · Variance Problem & Baselines

📉 High Variance in REINFORCE

  • REINFORCE has unbiased gradient estimates but extremely high variance — returns Gt can differ wildly between episodes even for the same policy.
  • High variance means the learning signal is noisy, requiring many episodes and small learning rates.
  • Solution: subtract a baseline b(s) from the return. This reduces variance without introducing bias.
REINFORCE with Baseline
θ ← θ + α · (Gt − b(st)) · ∇θ log π(at|st; θ)
b(st) — baseline (any function of state, typically V(st))
(Gt − b(st)) — advantage estimate: how much better was this action than average?
The baseline b(s) does not bias the gradient because E[b(s) · ∇ log π] = 0. The optimal baseline is the value function Vπ(s) — this is what Actor-Critic methods learn explicitly.
8 · The Advantage Function

⚡ A(s, a) — Relative Action Quality

  • The advantage function measures how much better action a is compared to the average action in state s.
  • It replaces the raw return in the policy gradient, centering updates around zero — actions better than average get a positive signal, worse ones get negative.
Advantage Function
Aπ(s, a) = Qπ(s, a) − Vπ(s)
Qπ(s, a) — expected return taking action a, then following π
Vπ(s) — expected return from state s following π (the baseline)
A > 0 — action a is better than average → increase its probability
A < 0 — action a is worse than average → decrease its probability
Policy Gradient with Advantage
∇θ J(θ) = Eπ[ Aπ(s,a) · ∇θ log π(a|s; θ) ]
9 · Function Approximation

🧠 Scaling with Neural Networks

  • For complex environments, π(a|s; θ) is represented by a deep neural network — maps raw states (pixels, sensor readings) to action probabilities.
  • Gradients are computed via backpropagation through the network using the policy gradient as the loss signal.
  • Key compatibility condition: the function approximator must be differentiable w.r.t. θ.
  • Common architectures: fully-connected for low-dimensional states, CNN for pixel inputs, RNN/LSTM for partial observability.
With function approximation, the policy is no longer guaranteed to converge to the exact optimum — but it can approximately converge to a local maximum of J(θ), which is usually sufficient in practice.
10 · Bias–Variance Tradeoff

⚖️ Choosing How to Estimate Returns

Monte Carlo (Gt)

Uses actual full-episode return. Unbiased but very high variance. Must wait for episode end.

1-Step TD (r + γV(s'))

Bootstraps using next-state value. Low variance but high bias (depends on accuracy of V). Can update online.

n-Step / GAE

Interpolates between MC and TD. Generalized Advantage Estimation (GAE) uses a λ parameter to tune the tradeoff.

Generalized Advantage Estimation (GAE)
ÂtGAE(γ,λ) = Σl=0∞ (γλ)l δt+l
δt = rt + γV(st+1) − V(st) — TD residual
λ=0 → pure 1-step TD (low variance, high bias)
λ=1 → pure Monte Carlo (high variance, low bias)
11 · Actor-Critic Architecture

🎭 Combining Policy & Value Learning

  • A natural extension: instead of using Monte Carlo returns, learn a critic V(s; w) to estimate the baseline/advantage online.
  • Actor — the policy network π(a|s; θ). Updated via policy gradient.
  • Critic — the value network V(s; w). Updated via TD error to minimize (r + γV(s';w) − V(s;w))².
  • The TD error δ = r + γV(s') − V(s) serves as a low-variance advantage estimate for the actor.
Actor Update (using critic's TD error as advantage)
θ ← θ + αθ · δ · ∇θ log π(a|s; θ)
Critic Update (TD learning)
w ← w + αw · δ · ∇w V(s; w)
δ = r + γV(s'; w) − V(s; w) — shared TD error used by both actor and critic
Actor-Critic is the foundation for modern algorithms: A2C, A3C, PPO, SAC, TD3 — all combine a learned policy with a learned value estimate.
12 · Pros & Cons

⚖️ Strengths & Limitations

Strengths
  • Works with continuous action spaces
  • Can learn stochastic policies
  • Smooth, stable optimization landscape
  • No need for argmax over actions
  • Strong theoretical guarantees (local optima)
Limitations
  • High variance (REINFORCE without baseline)
  • Sample inefficient — needs many trajectories
  • Sensitive to learning rate
  • May converge to local optima
  • On-policy: cannot reuse old experience

14 · Extensions & Modern Algorithms

🚀 Building on Policy Gradients

  • A2C / A3C — synchronous/asynchronous advantage actor-critic; multiple parallel workers for faster data collection.
  • PPO (Proximal Policy Optimization) — clips the policy update ratio to prevent destructively large steps. The most widely used modern algorithm.
  • TRPO (Trust Region Policy Optimization) — constrains the KL divergence between old and new policy to ensure monotonic improvement.
  • SAC (Soft Actor-Critic) — off-policy actor-critic with entropy regularization; highly sample-efficient for continuous control.
  • TD3 — deterministic policy gradient with twin critics and delayed updates; addresses overestimation in actor-critic.
