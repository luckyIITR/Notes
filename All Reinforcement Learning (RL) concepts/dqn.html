Deep Q-Network (DQN)

The algorithm that married Q-Learning with deep neural networks, enabling agents to master high-dimensional inputs like raw pixels. Introduced by DeepMind in 2013/2015.

1 · The Problem with Q-Tables

🧩 Why Q-Learning Alone Isn't Enough

  • Q-Learning stores one value per (state, action) pair in a table — this only works when the state space is small and discrete.
  • Real-world tasks (e.g. video games, robotics) have enormous or continuous state spaces — a 84×84 pixel Atari frame has ~1018,000 possible states.
  • A Q-table is completely infeasible. We need a way to generalize across similar states.
  • Solution: approximate Q(s,a) with a deep neural network parameterized by weights θ.
DQN = Q-Learning + Neural Network Function Approximation + two key stability tricks (Experience Replay and Target Network).
2 · Network Architecture

🧠 How the Network is Structured

  • Input: the current state s (e.g. stacked game frames preprocessed to 84×84 grayscale).
  • Output: one Q-value for every action simultaneously — avoids a separate forward pass per action.
  • For Atari: 3 convolutional layers → 2 fully-connected layers → output of size |A|.

3 · Two Key Innovations

💡 What Makes DQN Stable

Naively applying neural networks to Q-Learning is unstable — correlations in sequential data and a moving target cause divergence. DQN fixes this with two tricks.

🗄️ Experience Replay

Transitions (s, a, r, s') are stored in a replay buffer. During training, random mini-batches are sampled from this buffer, breaking the temporal correlation between consecutive updates.

🎯 Target Network

A separate, periodically-frozen copy of the network (θ⁻) is used to compute TD targets. This stabilizes training by preventing the target from shifting every step.

Without these two tricks, the network weights chase a constantly moving target computed by the same weights — leading to oscillation and divergence.
4 · Loss Function & Update

📉 Training Objective

DQN Loss (Mean Squared TD Error)
L(θ) = E[(y − Q(s, a; θ))²]
TD Target (computed with frozen target network θ⁻)
y = r + γ · maxa' Q(s', a'; θ⁻)
θ — online network weights (updated every step)
θ⁻ — target network weights (frozen, copied from θ every C steps)
y — TD target (treated as a fixed label during gradient step)
γ — discount factor
(s, a, r, s') — random mini-batch sampled from replay buffer
  • Gradient of the loss flows only through Q(s, a; θ) — the target y is treated as a constant.
  • Weights are updated via gradient descent (typically RMSProp or Adam).
5 · Experience Replay Buffer

🗄️ How the Buffer Works

  • A circular buffer of fixed capacity (e.g. 1 million transitions).
  • Every environment step, the tuple (s, a, r, s', done) is pushed in; oldest entries are overwritten when full.
  • At each training step, a random mini-batch (e.g. 32 or 64 transitions) is sampled uniformly.
  • Random sampling breaks temporal correlations — consecutive frames are highly correlated and would bias the network if trained in order.
  • Each transition can be replayed multiple times, improving data efficiency.
Replay also allows off-policy learning — stored transitions were generated by older versions of the policy, but that's fine because Q-Learning is off-policy by design.
6 · Target Network

🎯 Stabilizing the Training Target

  • Two identical networks: online network Q(s,a; θ) and target network Q(s,a; θ⁻).
  • The online network is updated every gradient step.
  • The target network is held frozen and only synced with the online network every C steps (e.g. every 1000 or 10000 steps).
  • TD targets are computed using θ⁻, so the target doesn't change on every step.
Think of the target network as a stable reference point. Without it, you'd be trying to hit a moving dartboard — the network update shifts Q(s,a), which immediately changes the target for the next update.
7 · Full Algorithm

⚙️ DQN Step-by-Step

  1. Initialize online network Q(s,a; θ) with random weights. Initialize target network Q(s,a; θ⁻) with θ⁻ = θ. Initialize replay buffer D with capacity N.
  2. For each episode, reset environment and observe initial state s (preprocessed).
  3. Select action a using ε-greedy on the online network: with prob ε pick random action, else argmaxa Q(s,a; θ).
  4. Execute action a, observe reward r, next state s', and done flag.
  5. Store transition (s, a, r, s', done) in replay buffer D.
  6. Sample a random mini-batch of transitions from D.
  7. Compute TD target: y = r if terminal, else y = r + γ · maxa' Q(s', a'; θ⁻).
  8. Perform gradient descent step on L(θ) = (y − Q(s,a; θ))² to update θ.
  9. Every C steps, copy online weights to target network: θ⁻ ← θ.
  10. Decay ε. Advance to s' and repeat from step 3.

9 · Pros & Cons

⚖️ Strengths & Limitations

Strengths
  • Scales to high-dimensional inputs (pixels)
  • End-to-end learning from raw observations
  • One network handles all actions efficiently
  • Replay improves sample efficiency
  • Demonstrated superhuman Atari performance
Limitations
  • Overestimates Q-values (maximization bias)
  • Discrete actions only (vanilla DQN)
  • High memory usage (large replay buffer)
  • Slow to train — millions of environment steps
  • Brittle to hyperparameter choices

11 · Extensions & Improvements

🚀 Building on DQN

  • Double DQN — decouples action selection and evaluation to fix overestimation bias.
  • Dueling DQN — splits the network into separate Value V(s) and Advantage A(s,a) streams.
  • Prioritized Experience Replay (PER) — samples transitions with high TD error more frequently.
  • Noisy Nets — replaces ε-greedy with learnable noise in network weights for exploration.
  • Distributional RL (C51) — models the full return distribution instead of just the expected value.
  • Rainbow DQN — combines all of the above into a single agent.
