Reinforcement Learning · Fundamentals

Introduction to Deep Reinforcement Learning

What RL is, how policy gradients work, and the core open problems — sparse rewards, credit assignment, and reward shaping.

1 · Why Reinforcement Learning?

🤖 The Robot Problem

In 2017, Peter Abeel gave a now-famous demo: a robot video showed a machine cleaning a room and fetching a beer. At the end he revealed the robot was entirely remote-controlled by a human operator.

The punchline is critical: modern robots are physically capable of a huge range of tasks. The bottleneck is purely software — intelligence. Creating useful robotics is not a hardware problem.

Getting a robot to pick up a bottle of beer is a hard AI problem, not a mechanics problem. Reinforcement learning is one of the most promising directions to close that gap.

The same lesson carries over from robotics to games: after the 2012 ImageNet breakthrough, supervised deep learning piled up wins in vision, speech, and text — but learning intelligent behaviour in complex, dynamic environments required something different.

2 · Supervised Learning vs. Reinforcement Learning

⚖️ The Core Difference

In supervised learning you feed an input to a model and you already know the correct output. A labelled dataset gives you a target for every sample, so you can compute gradients with back-propagation and directly minimise loss.

Applied to Pong: record a skilled human player for several hours, build a dataset of (screen frame → action taken), then train a small neural network to imitate those actions. It works — but it has two hard limits:

  • Data collection cost. You have to build the labelled dataset first, which is not always feasible.
  • Performance ceiling. The agent can never outperform the human it imitates. If your goal is superhuman play, supervised learning is structurally blocked.

In reinforcement learning there is no target label. The agent sees a frame, picks an action, and the only feedback it ever receives is a scalar reward signal — not a per-action correct answer.

PropertySupervised LearningReinforcement Learning
Feedback signalLabel for every inputReward after actions
Requires datasetYes — curated in advanceNo — experience is generated
Performance ceilingBounded by human labelsCan exceed human performance
Can exploreNoYes
3 · The Policy Network

🧠 Turning Frames Into Actions

In RL the neural network that maps observations to actions is called the policy network (often written π). For Pong it takes a raw screen frame as input and outputs — not a single action — but a probability distribution over actions:

Policy Output
π(a | s) = P(action = a | state = s)
π — policy network  ·  a — action (e.g. UP or DOWN)  ·  s — current state (screen frame)

During training the agent samples from this distribution rather than always picking the highest-probability action. Sampling ensures the agent explores the environment — trying different action sequences — instead of repeating the same behaviour forever.

The output is a probability, not a hard decision. This is what allows gradient-based learning: you can push the probability of good actions up and bad actions down.
4 · Policy Gradients

📈 Learning From Wins and Losses

Policy gradients are the simplest way to train a policy network. The idea is elegant: run the game, get a reward at the end, then nudge the network weights so that the actions taken during a good episode become more likely in the future — and less likely after a bad one.

The training loop:

  1. Initialise the policy network with random weights.
  2. Feed the current game frame through the network; sample an action from the output distribution.
  3. Send the action back to the game engine; receive the next frame.
  4. Repeat steps 2–3 until the episode ends (either player scores).
  5. Collect the reward: +1 if our agent scored, −1 if the opponent scored.
  6. For every action taken in the episode, compute the gradient that would make that action more likely.
  7. If the episode reward was positive → apply the gradient normally (increase probability).
    If the episode reward was negative → multiply the gradient by −1 (decrease probability).
  8. Update the network weights. Repeat from step 1.
Policy Gradient Update (REINFORCE)
∇J(θ) = 𝔼 [ ∇ log πθ(a|s) · R ]
θ — network weights  ·  πθ(a|s) — probability of taking action a in state s  ·  R — total episode reward (+ or −)

The minus-sign trick is the key insight: the same gradient computation works for both good and bad episodes. You just flip its sign to push the network away from the behaviour that led to a loss.

Strengths
  • No labelled dataset needed — the agent generates its own experience
  • Can in principle surpass human-level performance
  • Works with any differentiable neural network architecture
Limitations
  • Very sample-inefficient — needs vast amounts of experience
  • Entire episode is treated as uniformly good or bad (see below)
  • High variance in gradient estimates
5 · The Credit Assignment Problem

🎯 Which Actions Actually Mattered?

Imagine your Pong agent has been training for a while and is doing well — bouncing the ball back and forth. Near the very end of the episode it makes one mistake and lets the ball through: reward = −1.

Vanilla policy gradients treat every action in that episode as bad. The gradient update will decrease the probability of the defensive moves that were actually correct, not just the final blunder. This is the credit assignment problem: given a reward at the end of an episode, which specific actions in the sequence actually caused it?

In Pong, only the actions immediately before hitting the ball matter for the eventual score. Everything else — while the ball is mid-flight — is nearly irrelevant to the outcome.

The credit assignment problem is a direct consequence of the sparse reward setting: rewards arrive infrequently (once per episode), so the agent has to infer causal structure across a long sequence of actions with almost no signal.

6 · Sparse Rewards & Sample Inefficiency

🏜️ When Random Exploration Fails

Sample inefficiency is one of the central challenges in RL. Because rewards are sparse, the agent must generate enormous amounts of experience before it stumbles on a reward signal it can learn from.

In some environments sparse rewards don't just slow learning — they make it impossible without additional structure. Two canonical examples:

  • Montezuma's Revenge. The agent must navigate ladders, jump over a skull, grab a key, and then reach a door — in that order — to earn any reward. No single random action sequence is remotely likely to complete this chain. The policy gradient never sees a positive reward, so it has no gradient signal at all.
  • Robotic block stacking. A robot arm has ≈7 joints. The only reward is given when a block is successfully stacked. Random joint movements will never produce a successful stack, so the agent again sees zero signal.
Compare this with supervised vision: every input image has a label, so every forward pass produces a useful gradient. In sparse-reward RL most episodes produce zero gradient signal.
7 · Reward Shaping

🛠️ Manually Guiding the Policy

The traditional fix for sparse rewards is reward shaping: a human engineer manually designs a dense reward function that rewards intermediate behaviours on the path to the true goal.

For Montezuma's Revenge this might mean: give a small reward each time the agent avoids the skull, a separate reward for reaching the key, another for reaching the door, etc. The shaped reward function now provides gradient signal throughout the episode, and training converges much faster.

Why it helps
  • Converts a near-zero gradient signal into dense feedback
  • Makes convergence dramatically faster and more reliable
Why it's problematic
  • Not scalable. Every new environment needs its own custom reward function — for 57 Atari games that's 57 hand-crafted functions.
  • The alignment problem. Agents reliably find unexpected ways to maximise the shaped reward without achieving the intended goal. The policy overfits to the reward function, not to the desired behaviour.
  • Constrains potential. For games like AlphaGo, shaping the reward with human heuristics actively caps performance — the optimal policy may look nothing like human play.

Famous reward shaping failures:

  • An agent trained to jump was rewarded by the distance of its feet from the ground. It learned to grow a very tall body and do a backflip — maximising the metric without ever jumping normally.
  • Robotic control reward functions from published papers are sometimes hundreds of hand-tuned terms — a sign the problem is poorly specified, not solved.
The dilemma: sparse rewards make learning intractable, but shaped rewards introduce misalignment. Both paths have serious failure modes.
8 · Open Problems & What Comes Next

🔭 Where the Field Is Heading

The two core open problems in deep RL are directly connected: sample inefficiency and the sparse reward setting. Several research directions aim to tackle them without relying on hand-crafted reward functions:

  • Auxiliary tasks. Give the agent additional self-supervised objectives (e.g. predict the next frame, predict rewards) to learn richer representations from the same experience.
  • Intrinsic curiosity. Add an internal reward signal that encourages the agent to seek novel states — letting it explore meaningfully even when the extrinsic reward is zero.
  • Hindsight Experience Replay (HER). Relabel failed episodes retroactively as if the agent had actually intended to reach the state it ended up in — converting every trajectory into useful data.

Each of these approaches tries to give the agent a richer learning signal without a human specifying exactly what intermediate behaviours to reward — making RL more general and scalable.

Most RL breakthroughs — AlphaGo, Dota 1v1, robotic manipulation — are the product of significant engineering effort, not magic. Healthy scepticism of media hype is warranted; the field is genuinely advancing, but the hard problems remain hard.
