2 · The Environment (MDP)

🌍 What the Agent Sees and Does

CartPole is a Markov Decision Process. Every step: the environment gives the agent a state, the agent picks an action, and the environment returns a reward and a new state.

🕹️ Actions — 2 Discrete Choices

  • 0 → Push cart left (−10 N force)
  • 1 → Push cart right (+10 N force)
Reward & Termination
r = +1 per surviving step  ·  max return = 500
Episode ends when:  |θ| > 12° (pole falls)  or  |x| > 2.4 (cart off track)  or  steps = 500 (solved!)
Because the reward is +1 every step that the pole is still up, the agent only needs to learn one thing: keep the pole vertical as long as possible.
3 · DQN — The Decision Network

🧠 Animated Neural Network

The DQN reads the 4-dimensional state and outputs two Q-values — one per action. The action with the higher Q-value is chosen. Watch the forward-pass signal flow from inputs to outputs.

Pulses animate every forward pass  ·  ● chosen action  ·  ● rejected action

⚙️ Why This Works

  • The network approximates Q(s, a) — the expected total future reward from state s taking action a.
  • A target network (frozen copy) provides stable TD targets and prevents the training from chasing a moving goal.
  • CartPole only needs 4 → 128 → 128 → 2. A tiny network that converges in ~250 episodes.
Bellman TD Target & Loss
y = r + γ · maxa′ Qθ̄(s′, a′) · (1 − done)
ℒ(θ) = 𝔼 [ (y − Qθ(s, a))² ]
θ̄ = frozen target network  ·  γ = 0.99 (discount)  ·  done = 1 if terminal

🔀 Why Random Sampling Matters

  • Sequential experience is highly correlated — training on it directly causes the network to overfit to recent behavior and forget earlier lessons.
  • Random sampling breaks this correlation, giving the network a diverse mix of early failures and recent successes in every batch.
  • The circular buffer (size 10 000) keeps older, diverse experiences alive — the agent learns from all its history, not just the last few episodes.
Experience replay is why DQN can use standard supervised-learning SGD. Without it, the correlations between consecutive steps make gradient descent wildly unstable.
5 · ε-Greedy Exploration

🎲 The Explore–Exploit Tradeoff

Early in training the Q-network outputs garbage — acting greedily would mean repeating the same wrong action. ε-greedy forces random exploration that decays over time as the network improves.

ε Decay Schedule
εt = εmin + (1 − εmin) · e−t / τ
εmin = 0.01  ·  τ = 500 steps  ·  At step 0: ε=1.0 → fully random. At step 2000: ε≈0.02.

🛠️ Key Hyperparameters

Learning Rate — 1e-3

Adam optimizer. Too high (>5e-3) causes divergence; too low is unnecessarily slow.

Replay Buffer — 10 000

CartPole episodes are short, so the buffer fills fast and stays diverse.

Target Sync — every 10 ep.

Syncing too often collapses stability; too rarely makes targets stale.

Discount γ = 0.99

Episodes can run 500 steps — a high γ correctly values long-horizon survival.

7 · Training Loop

🔄 The Algorithm, Step by Step

  1. Reset the environment → receive initial state s
  2. With probability ε pick a random action (explore); otherwise pick argmax Q(s, a) from the network (exploit)
  3. Execute action a → receive reward r, next state s′, and done flag
  4. Store (s, a, r, s′, done) in the replay buffer
  5. If buffer has ≥ 64 transitions: sample a random mini-batch, compute TD targets using the frozen target network, and take a gradient step to minimise the MSE loss
  6. Every 10 episodes, copy the online Q-network weights into the target network
  7. Repeat until the 100-episode rolling average reward ≥ 475 → Solved!
Full Implementation (PyTorch + Gymnasium)
import torch, torch.nn as nn, torch.optim as optim
import numpy as np, gymnasium as gym, random
from collections import deque

# ── Network ────────────────────────────────────────────
class QNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(4, 128), nn.ReLU(),
            nn.Linear(128, 128), nn.ReLU(),
            nn.Linear(128, 2))
    def forward(self, x): return self.net(x)

# ── Replay Buffer ───────────────────────────────────────
class Buffer:
    def __init__(self): self.d = deque(maxlen=10_000)
    def push(self, *t): self.d.append(t)
    def sample(self, n):
        s,a,r,ns,d = zip(*random.sample(self.d, n))
        return (torch.FloatTensor(np.array(s)),
                torch.LongTensor(a), torch.FloatTensor(r),
                torch.FloatTensor(np.array(ns)), torch.FloatTensor(d))
    def __len__(self): return len(self.d)

# ── Training ────────────────────────────────────────────
env    = gym.make("CartPole-v1")
q, qt  = QNet(), QNet()
qt.load_state_dict(q.state_dict())
opt, buf = optim.Adam(q.parameters(), lr=1e-3), Buffer()
γ, BATCH, steps = 0.99, 64, 0

for ep in range(600):
    s, _ = env.reset(); total = 0
    while True:
        ε = 0.01 + 0.99 * np.exp(-steps / 500)
        a = random.randint(0,1) if random.random() < ε else \
            q(torch.FloatTensor(s).unsqueeze(0)).argmax().item()
        ns, r, term, trunc, _ = env.step(a)
        done = term or trunc
        buf.push(s, a, r, ns, float(done)); steps += 1
        if len(buf) >= BATCH:
            S,A,R,NS,D = buf.sample(BATCH)
            tgt = R + γ * qt(NS).max(1).values * (1-D)
            loss = nn.functional.mse_loss(q(S).gather(1,A.unsqueeze(1)).squeeze(), tgt.detach())
            opt.zero_grad(); loss.backward(); opt.step()
        s, total, _ = ns, total + r, done and None
        if done: break
    if ep % 10 == 0: qt.load_state_dict(q.state_dict())
    if ep >= 99 and np.mean(rewards[-100:]) >= 475:
        print(f"Solved at episode {ep}!"); break

8 · What the Agent Actually Learns

🔍 The Emergent Policy

The trained Q-network discovers a simple but robust control law entirely from reward signals — no physics equations provided:

  • If the pole is tilting right (θ > 0), push right to chase the hinge back under it.
  • Angular velocity θ̇ matters more than position — the network learns to weight it heavily for anticipatory corrections.
  • When the cart drifts to an edge, it gradually shifts priority from pole correction to cart centering.
The Q-network discovers the same corrective reflex a control engineer would derive with PD control — purely from reward signals and gradient descent.

🚀 After CartPole — Natural Progression

  • MountainCar — sparse rewards; the agent must learn to swing the car back-and-forth to build momentum before it can reach the goal.
  • LunarLander — more complex 8D state + shaped reward; tests PPO's ability to balance multiple simultaneous objectives.
  • Atari — pixel observations; requires a CNN encoder + large replay buffer (the original DQN paper, 2015).
  • MuJoCo continuous tasks — continuous action spaces; switch to SAC or TD3.
