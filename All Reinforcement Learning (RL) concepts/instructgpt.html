LLM Alignment · RLHF · Instruction Following

Training Language Models to Follow Instructions with Human Feedback

The paper that created ChatGPT-style models — applying a 3-step RLHF pipeline (SFT → Reward Model → PPO) to align GPT-3 with human intent. The resulting model (InstructGPT) is preferred over GPT-3 175B despite being 100× smaller.

Training language models to follow instructions with human feedback (InstructGPT)
Long Ouyang, Jeff Wu, Xu Jiang, Diogo Almeida, Carroll Wainwright, Pamela Mishkin, Chong Zhang, Sandhini Agarwal, Katarina Slama, Alex Ray, John Schulman, Jacob Hilton, Fraser Kelton, Luke Miller, Maddie Simens, Amanda Askell, Peter Welinder, Paul Christiano, Jan Leike, Ryan Lowe  ·  NeurIPS 2022  ·  OpenAI
1 · The Problem with Base Language Models

🧩 Why GPT-3 Isn't Helpful by Default

  • Base LLMs like GPT-3 are trained on a next-token prediction objective — they learn to predict what text comes next on the internet, not to be helpful, honest, or harmless.
  • The training objective is fundamentally misaligned with what users actually want.
  • Base GPT-3 is prone to: ignoring instructions, making up facts, generating toxic content, and producing verbose but unhelpful outputs.
GPT-3 base (unaligned)

"Explain the moon landing." → Continues with more questions, irrelevant trivia, or a different topic entirely — it's continuing text, not answering.

InstructGPT (aligned) ✓

"Explain the moon landing." → Gives a direct, clear, well-structured explanation of the Apollo 11 mission, addressing what the user actually wanted.

The core thesis: making LLMs "aligned" means making their behavior match human intent — being helpful, honest, and avoiding harm. This requires a different training signal than next-token prediction.
2 · The Three-Step RLHF Pipeline

🏗️ Overview of InstructGPT Training

Step 1
SFT
Supervised Fine-Tuning

Fine-tune GPT-3 on ~13K high-quality human-written demonstrations of ideal responses to diverse prompts.

Step 2
RM
Reward Model Training

Collect ~33K human preference rankings of model outputs. Train a reward model to predict which output humans prefer.

Step 3
PPO-ptx
RL Fine-Tuning

Use PPO to optimize the SFT model against the reward model, with a KL penalty to prevent it straying too far from the SFT baseline.

4 · Step 1 — Supervised Fine-Tuning (SFT)

✍️ Teaching the Model to Be a Good Assistant

  • Labelers write ideal responses to a sampled prompt — demonstrating what a helpful, honest, and harmless reply looks like.
  • GPT-3 is fine-tuned on these (prompt, demonstration) pairs using standard supervised next-token prediction loss.
  • Training: 16 epochs, cosine LR decay, residual dropout 0.2. Model checkpointed by RM score on validation set.
  • The SFT model becomes the starting point for both reward model training and PPO fine-tuning.
Prompt (from API user)
"Explain the concept of overfitting in machine learning to a 10-year-old."
Labeler-written ideal response
"Imagine you're studying for a spelling test by memorizing only the exact words from last year's test. You do great on those words, but when the teacher gives you new words, you fail — because you memorized instead of learning the pattern. That's overfitting: a model learns the training data too perfectly and can't handle new examples."
5 · Step 2 — Reward Model Training

🏆 Learning to Score Outputs Like a Human

  • For each prompt, the SFT model generates 4–9 output responses. Labelers rank all of them from best to worst.
  • Each ranking is decomposed into K choose 2 pairwise comparisons (e.g. 4 outputs → 6 pairs).
  • A reward model — initialized from the 6B SFT model with a scalar head replacing the final unembedding layer — is trained to predict which output is preferred.
Reward Model Loss (Pairwise Ranking)
L(ψ) = − 1/(K choose 2) · E(x,yw,yl)∈𝒟 [ log σ( rψ(x, yw) − rψ(x, yl) ) ]
x — prompt
yw — the preferred (winning) response
yl — the less-preferred (losing) response
rψ(x, y) — scalar reward score from the reward model
σ — sigmoid function
Maximizes the gap between preferred and dispreferred scores.
  • Using K outputs per prompt and all pairwise comparisons is much more label-efficient than collecting one comparison per prompt.
  • The 6B parameter RM (not 175B) was found to be more stable and avoid overfitting with the available labels.
6 · Step 3 — PPO Fine-Tuning (PPO-ptx)

🔁 Optimizing the Policy Against the Reward Model

  • The SFT model (now the RL policy πθ) is further fine-tuned using PPO.
  • For each prompt, the policy generates a response. The reward model scores it. PPO updates θ to increase the probability of high-scoring responses.
  • A critical KL penalty term prevents the PPO model from diverging too far from the SFT baseline — preventing reward hacking and mode collapse.
  • A pretraining term (ptx) is added to preserve GPT-3's general capabilities — the "alignment tax" problem.
Full PPO-ptx Reward / Objective
r(x, y) = rψ(x, y) − β · log [ πθ(y|x) / πSFT(y|x) ]
rψ(x, y) — score from the reward model (higher = more preferred)
β · KL[ πθ ‖ πSFT ] — penalty for deviating from SFT model (β ≈ 0.02)
πSFT — the frozen SFT model used as reference
KL term prevents reward hacking — policy can't exploit weaknesses in rψ without being penalized
Final Training Objective (PPO-ptx)
L(θ) = E(x,y)~πθ[ r(x,y) ] + γ · Ex~𝒟pretrain[ log πθ(x) ]
γ · E[ log πθ(x) ] — pretraining loss on original GPT-3 data (preserves general capabilities)
γ — pretraining weight (typically very small, e.g. 27.8)
The "ptx" suffix denotes this additional pretraining gradient mix-in.
7 · Full Training Algorithm

⚙️ Step-by-Step

  1. Sample a diverse set of prompts from the OpenAI API prompt distribution. Have human labelers write ideal responses. Fine-tune GPT-3 on these (prompt, response) pairs → SFT model.
  2. Sample more prompts. For each, generate 4–9 responses from the SFT model. Have labelers rank all responses from best to worst.
  3. Decompose each set of K rankings into all K(K-1)/2 pairwise comparisons. Train reward model rψ (6B params) on these pairs using the ranking loss.
  4. Initialize the PPO policy πθ from the SFT model. Freeze the SFT model as the KL reference πSFT.
  5. For each PPO iteration: sample a prompt from the PPO dataset, generate a response with πθ, score it with rψ, compute the full reward r = rψ − β·KL.
  6. Update πθ with PPO gradient ascent on the full reward plus the pretraining mix-in loss.
  7. Repeat steps 5–6 for the full PPO training budget. The final model is InstructGPT.
8 · Labeler Guidelines — The 3 H's

📋 What Labelers Were Instructed to Optimize For

🤝 Helpful

Follow the user's explicit instructions. Infer intent when ambiguous. Be concise, direct, and actually answer the question. Avoid unnecessary caveats or padding.

✅ Honest

Don't fabricate facts or make up citations. Express appropriate uncertainty. Don't say what you think the user wants to hear if it's not true.

🛡️ Harmless

Avoid generating content that is racist, sexist, toxic, or dangerous. Decline requests that would cause real-world harm.

Helpfulness was ranked first — an overly cautious model that refuses too many requests is also considered misaligned. The goal is genuine usefulness, not just safety-washing.

10 · The Alignment Tax

💸 Capability Regression from Alignment

  • Fine-tuning with RLHF can cause a model to forget general capabilities it had after pretraining — known as the alignment tax.
  • InstructGPT showed small but measurable regression on public NLP benchmarks like SQuAD, HellaSwag, and WMT translation tasks.
  • The PPO-ptx fix — mixing in gradients from the original pretraining data — substantially reduced this regression.
  • This tension between alignment and capability is an open research problem, relevant to all subsequent fine-tuned models.
The alignment tax motivated continued research into techniques like DPO (Direct Preference Optimization) which avoids PPO entirely and shows less capability regression in some settings.
11 · Limitations & Failure Modes

⚠️ What InstructGPT Still Gets Wrong

Remaining Issues
  • Still hallucinates facts confidently
  • Gives long answers when short ones are better
  • Assumes false premises in questions
  • Sensitive to prompt wording
  • Can fail on complex reasoning chains
  • Reflects labeler demographic biases
Systemic Concerns
  • Labeler values ≠ all of humanity
  • Reward model can be gamed (reward hacking)
  • Model "aligned" to labeler preferences, not ground truth
  • May be overly cautious, refusing benign requests
  • Alignment doesn't transfer across languages well

14 · Legacy & What Came Next

🌍 Impact on AI Development

  • ChatGPT (Nov 2022) — same 3-step pipeline applied to GPT-3.5. Became the fastest-growing consumer product in history.
  • GPT-4 (Mar 2023) — same RLHF methodology at larger scale with additional safety work.
  • Claude (Anthropic, 2023) — extends RLHF with Constitutional AI (CAI), replacing human labels with AI-generated critiques based on a written constitution.
  • Llama 2-Chat, Mistral Instruct — open-source models using the same SFT+RLHF pipeline.
  • DPO (2023) — Direct Preference Optimization eliminates the reward model entirely, deriving the optimal policy directly from preference data with a simpler loss function.
  • GRPO (2024) — Group Relative Policy Optimization removes the critic network, using group-normalized rewards. Used to train DeepSeek-R1.
Every major instruction-following AI assistant deployed today — ChatGPT, Claude, Gemini, Llama-chat — traces its alignment methodology directly back to this paper.
