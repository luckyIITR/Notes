Q-Learning

A foundational off-policy temporal-difference algorithm that learns the optimal action-value function directly, regardless of the policy being followed.

1 · What is Q-Learning?

💡 Intuition

  • Q-Learning learns a Q-function — the expected cumulative reward of taking action a in state s and then acting optimally.
  • It is model-free: no knowledge of transition probabilities or reward dynamics is needed.
  • It is off-policy: the agent can follow any exploratory behavior policy (e.g. ε-greedy) while learning the optimal greedy policy simultaneously.
  • Introduced by Chris Watkins in 1989. One of the most widely studied RL algorithms.
2 · The Q-Value

📌 Q(s, a) — Action-Value Function

  • Q(s, a) estimates the total discounted return starting from state s, taking action a, and following the optimal policy thereafter.
  • Stored in a Q-table for discrete state/action spaces (rows = states, columns = actions).
  • Replaced by a neural network (DQN) for large or continuous spaces.
Definition
Q*(s, a) = E [ Σ γᵗ rₜ | s₀=s, a₀=a, π* ]
The optimal policy is simply π*(s) = argmaxa Q*(s, a) — always pick the action with the highest Q-value.
3 · Update Rule

🔄 Bellman-Based TD Update

Q-Learning Update
Q(s,a) ← Q(s,a) + α [ r + γ · maxa' Q(s',a') − Q(s,a) ]
α — learning rate (how fast to update)
γ — discount factor (importance of future rewards)
r — immediate reward received
s' — next state after taking action a
maxa' Q(s',a') — best possible Q-value from the next state
[ r + γ·max Q(s',a') − Q(s,a) ] — TD error (δ)
The key insight: Q-Learning always bootstraps using the greedy max over next-state Q-values — this is what makes it off-policy. The behavior policy (which action you actually took) does not affect the target.
4 · Algorithm

⚙️ Step-by-Step

  1. Initialize Q(s, a) arbitrarily for all s, a (e.g. all zeros). Set Q(terminal, ·) = 0.
  2. For each episode, reset to starting state s.
  3. Choose action a using a behavior policy derived from Q (e.g. ε-greedy).
  4. Take action a, observe reward r and next state s'.
  5. Apply the update: Q(s,a) ← Q(s,a) + α [ r + γ · maxₐ' Q(s',a') − Q(s,a) ]
  6. Set s ← s'. Repeat from step 3 until terminal state.
  7. Repeat episodes until Q converges.
5 · Exploration

🔍 ε-Greedy Strategy

  • With probability ε: take a random action (explore).
  • With probability 1 − ε: take argmaxa Q(s,a) (exploit).
  • ε-decay: start with high ε (explore more early on) and reduce it over time as Q stabilizes.
Exploration is needed so the agent visits all state-action pairs enough times for Q to converge. Without it, Q may never discover better actions.
6 · Convergence

📈 When Does Q-Learning Converge?

  • Q-Learning converges to Q* (the optimal Q-function) with probability 1, given:
  • All state-action pairs are visited infinitely often.
  • Learning rate α satisfies the Robbins-Monro conditions: Σ αₜ = ∞ and Σ αₜ² < ∞.
  • The environment is a stationary MDP (transition/reward functions don't change).
In practice, a fixed small α (e.g. 0.1) works well for most problems, even though it doesn't satisfy the strict theoretical conditions.

8 · Pros & Cons

⚖️ Strengths & Limitations

Strengths
  • Simple to implement
  • Guaranteed convergence to Q*
  • Off-policy — can learn from any experience
  • No environment model needed
Limitations
  • Q-table doesn't scale to large state spaces
  • Overestimates Q-values (maximization bias)
  • Slow to converge in sparse-reward environments
  • Struggles with continuous action spaces
9 · Q-Learning vs SARSA

🔀 Key Difference

Q-LearningSARSA
TypeOff-policyOn-policy

SARSA uses the actual next action taken (a'), while Q-Learning uses the best possible next action. This single difference defines on-policy vs off-policy.
10 · Extensions

🚀 What Comes Next

  • Double Q-Learning — fixes maximization bias by decoupling action selection and evaluation.
  • DQN — replaces Q-table with a deep neural network; adds experience replay and a target network.
  • Dueling DQN — separates V(s) and advantage A(s,a) streams.
  • Prioritized Experience Replay — samples transitions with high TD error more frequently.
