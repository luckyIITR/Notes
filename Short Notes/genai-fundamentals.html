Gen AI Fundamentals — Mind Map

The whole Gen AI fundamentals course, distilled into scannable branches: architecture → prompting → training → fine-tuning → alignment → deployment → apps.

1 · Foundations

🌐 What Gen AI Is

Gen AISubset of ML; learns statistical patterns from massive human text.
Foundation / base modelsBillions of parameters (= memory); trained on trillions of words.
Emergent propertiesReasoning, task breakdown — beyond pure language.
General-purpose techLike electricity/deep learning — useful across the economy.
A small fine-tuned model can rival a giant on a focused task — bigger isn't always needed.
2 · Talking to a Model

💬 Core Vocabulary

PromptThe text you pass in.
Context windowMemory available for the prompt (few thousand words).
InferenceThe act of generating text.
CompletionOutput = prompt + generated text.
3 · Transformer Architecture

⚡ Attention Is All You Need (2017)

Replaced RNNs (poor at long context, hard to scale). Wins: scales on GPUs, parallel processing, learns to pay attention to word meaning.

Self-attentionWeighs every word vs. every other word; weights learned in training.
Multi-head attention12–100 heads, each learns a different language aspect, in parallel.
EncoderEncodes input into a deep representation.
DecoderGenerates tokens in a loop until a stop token.
1 Tokenize→ 2 Embedding→ 3 Positional encoding→ 4 Self-attention→ 5 Feed-forward + softmax → logits
4 · Model Variants

🪢 Three Families

Encoder-only (autoencoding)Masked LM, bi-directional. Classification, NER. → BERT, RoBERTa.
Decoder-only (autoregressive)Causal LM, uni-directional. Text gen, strong zero-shot. → GPT, BLOOM, LLaMA.
Encoder-decoder (seq2seq)Span corruption (T5). Translation, summarization, Q&A. → T5, BART.
Rule of thumbLarger models of any type generalize better with less help.
5 · Prompting & In-Context Learning

✍️ Teach by Example

Zero-shotInstruction only. Best for the largest models.
One-shotOne example + input. For smaller models.
Few-shotSeveral mixed examples. For even smaller models.
Prompt engineeringIteratively revise wording to get desired behavior.
Mind the context window: if ~5–6 examples still fail → fine-tune instead.
6 · Inference Config

🎛️ Controlling Output

Max new tokensCaps generated length (a limit, not a guarantee).
GreedyAlways highest-prob word; repetitive.
Random samplingWeighted by probability; less repetition.
Top-k / Top-pSample from k best, or from cumulative prob ≤ p (nucleus).
Temp < 1 peaked → less random Temp = 1 default Temp > 1 flatter → more creative
Temperature actually alters the distribution; top-k/top-p only restrict sampling.

8 · Pre-training & Memory

💾 Training Cost

Pre-trainingSelf-supervised on vast unstructured text; only 1–3% of tokens survive curation.
Memory math1 param = 4 bytes (FP32). 1B params = 4 GB weights; ~24 GB to train (≈6×).
Model cardsOn HF/PyTorch hubs — best uses, training, limits.
CUDANVIDIA libs PyTorch/TF use for matrix math.
9 · Quantization

📉 Precision vs. Memory

FP324 bytes — default, full precision.
FP162 bytes — some precision loss.
BF162 bytes — keeps FP32 range, stable; popular pick.
INT81 byte — big precision loss.
Quantization shrinks footprint but param count stays the same.
10 · Distributed Training

🖧 Across Many GPUs

DDPFull model copied per GPU; needs to fit on one GPU.
FSDPShards params/grads/optimizer states (based on ZeRO).
ZeRO-1 optimizer → 4× ZeRO-2 + grads → 8× ZeRO-3 + params → scales with GPUs

12 · Instruction Fine-Tuning

🧭 Follow Instructions

Supervised training on labeled prompt–completion pairs → an instruct model. Full fine-tuning = all weights updated (same cost as pre-training).

Catastrophic forgettingSingle-task tuning degrades other abilities. Fix: multitask tuning or PEFT.
FLAN-T5: tuned on 473 datasets / 146 task categories; SAMSum & dialogsum for dialogue summarization.
13 · PEFT & LoRA

🪶 Cheap Fine-Tuning

PEFTTrain a small subset, freeze the rest. Single GPU, robust to forgetting, swap per task.
SelectiveTune some existing params.
ReparameterizationLow-rank matrices → LoRA.
AdditiveAdapter layers; soft prompts (prompt tuning).
LoRAInject rank-decomp matrices on attention layers; ranks 4–32 (plateau past 16). ~86% fewer params. QLoRA = + quantization.
Prompt tuningTrain 20–100 virtual tokens, weights frozen. Matches full FT at ~10B+ params.
14 · Evaluation

📐 Metrics & Benchmarks

ROUGESummarization — uni/bi-gram & LCS overlap vs. reference.
BLEUTranslation — avg n-gram precision.
Diagnostic onlyClipping fixes repetition; never the sole final metric.
BenchmarksGLUE, SuperGLUE, MMLU, BIG-bench, HELM (adds fairness/bias/toxicity).
15 · Alignment & RLHF

🤝 Helpful · Honest · Harmless

RLHF aligns the LLM with human preferences via RL. Mapping: agent = LLM, action = next token, environment = context window, reward = human-preference alignment.

Prompt → LLM completion→ Reward model scores it→ PPO updates weights→ repeat → aligned LLM
Reward modelHumans rank completions → pairwise loss −log σ(rj−rk) → binary classifier (no humans after).
PPOSmall, bounded updates in a trust region; uses advantage + value + entropy loss.
Reward hackingModel games the metric. Fix: frozen reference model + KL-divergence penalty.
Constitutional AI / RLAIFRules-based self-critique replaces human feedback (Anthropic).
Alternatives to PPO: Q-learning and the simpler DPO. Combine RLHF + PEFT to reuse one base model and halve memory.
16 · Deployment Optimization

📦 Smaller, Faster

DistillationTeacher trains a smaller student; best for encoder-only.
PTQQuantize to 16/8-bit after training (needs calibration).
PruningDrop near-zero weights; may need retraining.
17 · Limitations & Augmentation

⚠️ What Training Can't Fix

Knowledge cutoffFrozen at pre-training time.
Weak mathPredicts tokens, doesn't compute.
HallucinationConfidently invents answers.
OrchestrationLangChain connects model to data & APIs at runtime.
18 · RAG

📚 External Knowledge

Query encoder→ Search vector store→ Expanded prompt→ LLM completion
ChunksSplit long sources to fit the context window.
Vector store / DBEmbed + cosine similarity; DB adds keys → citations.
Fixes knowledge cutoff & hallucination far cheaper than retraining.
19 · Reasoning & Agents

🧩 LLM as Reasoning Engine

Chain-of-Thought (CoT)Include intermediate reasoning steps in examples → step-by-step answers.
PALLLM writes a Python script; interpreter runs it for reliable math.
ReActLoop of thought → action → observation; actions from a fixed set (search/lookup/finish).
OrchestratorAutomates hand-offs to tools, APIs, interpreters.
A limited action set is critical — LLMs are creative and may propose impossible steps.
20 · LangChain & App Architecture

🏛️ Full Stack

LangChainPrompt templates, memory, tools; connect into chains; agents for PAL/ReAct.
The stackInfrastructure → models → external sources → tools/frameworks → interface & security.
Prefer large models for reasoning; later fine-tune a smaller one on collected data.
