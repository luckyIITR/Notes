ML Zoomcamp · Session 2

Machine Learning for Regression

A hands-on project: predict car prices with linear regression — from data prep and EDA to a validation framework, the normal equation, and a baseline model.

1 · The Project

🏷️ Predicting Car Prices

Back to the scenario from the introduction: help a seller on a car classifieds website pick the best price. In this session we build that model for real, using a Kaggle dataset of car prices.

The dataset holds many characteristics of cars — make, model, year, engine type, fuel type, transmission, and so on — plus the target column MSRP (Manufacturer Suggested Retail Price), which is the price we want to predict.

The full plan: get the data → EDA → train linear regression → implement it ourselves → evaluate with RMSE → feature engineering → fix numerical issues with regularization → use the model.
2 · Data Preparation

🧹 Cleaning & Normalizing

Download the CSV and load it with Pandas, then inspect the first rows. The column names are inconsistent — mixed case, some with spaces, some with underscores — which makes them awkward to access (a space breaks dot notation).

import pandas as pd

df = pd.read_csv("data.csv")
df.head()

# normalize column names: lowercase + underscores
df.columns = df.columns.str.lower().str.replace(" ", "_")

The string values have the same problem (sometimes all caps, sometimes not). First find the string columns via dtypes (type object), then normalize each one the same way.

strings = list(df.dtypes[df.dtypes == "object"].index)

for col in strings:
    df[col] = df[col].str.lower().str.replace(" ", "_")
The columns attribute is an Index, which (like a Series) has a .str accessor — so the same string methods work on column names and on column values alike.
3 · Exploratory Data Analysis

🔍 Understanding the Data

Loop over the columns to get a feel for what's there — the first few values, the unique values, and how many unique values each column has.

for col in df.columns:
    print(col)
    print(df[col].unique()[:5])
    print(df[col].nunique())
    print()

This reveals the mix of categorical columns (make, model, fuel type, transmission…) and numerical ones (engine HP, cylinders, MPG, popularity, MSRP). Popularity is interesting — the dataset authors counted Twitter mentions, the idea being that more popular cars are mentioned more.

📊 The Price Distribution & Long Tail

Plot the price with a histogram using Matplotlib + Seaborn. Most cars are cheap, but a few are extremely expensive — a long-tail distribution.

import matplotlib.pyplot as plt
import seaborn as sns
%matplotlib inline

sns.histplot(df.msrp, bins=50)
sns.histplot(df.msrp[df.msrp < 100000], bins=50)   # zoom in

Long tails confuse models. The fix is to apply a logarithm, which compresses large values. To avoid log(0) (undefined), add 1 first — NumPy's log1p does exactly this.

Log Transform
ylog = log(y + 1) = np.log1p(y)
Turns the long-tail shape into something close to a normal distribution (bell curve) — far easier for models to predict.
import numpy as np

log_price = np.log1p(df.msrp)
sns.histplot(log_price, bins=50)   # tail gone, bell-shaped
Models predict a target far better when it's roughly normally distributed. Long-tail targets (common for prices) should usually be log-transformed first.

🕳️ Missing Values

Some cells are NaN (missing). Count them per column — we'll have to handle these before training.

df.isnull().sum()   # missing count per column
4 · Setting Up Validation

✂️ Train / Validation / Test Split

As covered in Session 1, split the data into three parts (60 / 20 / 20). We do it manually with plain NumPy and Pandas. Compute the sizes, deriving the train size by subtraction so rounding never drops a record.

n = len(df)
n_val = int(0.2 * n)
n_test = int(0.2 * n)
n_train = n - n_val - n_test   # subtract to avoid rounding loss

The data is ordered (all BMWs together, etc.), so we must shuffle before splitting — otherwise a class could land entirely in one split. Use a fixed seed for reproducibility.

idx = np.arange(n)
np.random.seed(2)
np.random.shuffle(idx)

df_train = df.iloc[idx[:n_train]]
df_val   = df.iloc[idx[n_train:n_train + n_val]]
df_test  = df.iloc[idx[n_train + n_val:]]

df_train = df_train.reset_index(drop=True)
df_val   = df_val.reset_index(drop=True)
df_test  = df_test.reset_index(drop=True)

Apply the log transform to the target for each split, store it as a NumPy array, then delete the target column from the feature frames.

y_train = np.log1p(df_train.msrp.values)
y_val   = np.log1p(df_val.msrp.values)
y_test  = np.log1p(df_test.msrp.values)

del df_train["msrp"]
del df_val["msrp"]
del df_test["msrp"]
Always remove the target from the feature frame after splitting. Leaving it in means you might accidentally train on the answer — the model looks perfect and you waste hours debugging.
5 · Linear Regression — One Car

📈 The Model for a Single Observation

Linear regression solves regression problems — it predicts a number. For one car described by a feature vector xi = (xi1, xi2, …, xin), the prediction combines a bias term with a weighted sum of the features.

Linear Regression (single observation)
g(xi) = w0 + Σj wj · xij
w0 — the bias term: the baseline prediction when we know nothing about the car.
wj — the weight for feature j; each feature contributes its value × its weight. 
def linear_regression(xi):
    n = len(xi)
    pred = w0
    for j in range(n):
        pred = pred + w[j] * xi[j]
    return pred

# example: engine_hp, city_mpg, popularity
xi = [453, 11, 86]
w0 = 7.17
w  = [0.01, 0.04, 0.002]

Interpreting the parts: the bias (7.17) is the baseline; each extra horsepower adds 0.01, each extra city-MPG adds 0.04, and popularity barely moves the price. The result (~12.3) is in log space, so undo the transform to get dollars.

Undo the Log
price = exp(pred) − 1 = np.expm1(pred)
expm1 is the inverse of log1p — it exponentiates and subtracts 1 in one step.
6 · Linear Regression — Vector Form

🧮 From a Loop to Matrix Multiplication

The weighted sum is just a dot product, so the model simplifies to xiTw. To absorb the lone bias term, add a virtual feature that is always 1 as the first element of every feature vector, with w0 as its weight.

Vector & Matrix Forms
g(xi) = xiTw   →   predictions = X · w
Prepend a column of 1s to X (the bias feature) so X is m × (n+1) and w is length n+1.
Applying the model to every car is just matrix × vector multiplication. 
X = np.array([
    [1, 148, 24, 1385],
    [1, 132, 25, 2031],
    [1, 453, 11, 86],
])
w_new = [7.17, 0.01, 0.04, 0.002]   # bias first

predictions = X.dot(w_new)   # one prediction per row
The progression: a for loop → recognizing a dot product → adding a fictional 1-feature to fold in the bias → generalizing to the whole feature matrix as X · w.
7 · Training — The Normal Equation

🎯 Where Do the Weights Come From?

We want X · w ≈ y. Ideally w = X-1y, but X is rectangular (many rows, few columns) so it has no inverse. The trick: multiply by XT to form the square Gram matrix XTX, which is invertible, then solve.

Normal Equation
w = (XTX)-1 XT y
Not the exact solution (none exists) — the closest possible one (least squares).
The Gram matrix XTX is (n+1) × (n+1), square, so its inverse usually exists. 
def train_linear_regression(X, y):
    ones = np.ones(X.shape[0])
    X = np.column_stack([ones, X])      # add bias column

    XTX = X.T.dot(X)                    # Gram matrix
    XTX_inv = np.linalg.inv(XTX)
    w_full = XTX_inv.dot(X.T).dot(y)    # normal equation

    return w_full[0], w_full[1:]        # bias, weights
Adding the bias column of 1s happens inside the function, so callers never worry about it. For the derivation/proof, see The Elements of Statistical Learning.
8 · The Baseline Model

🏗️ First Model From Numerical Features

For a baseline, use only the numerical columns. Extract them, fill missing values with 0, and train.

base = ["engine_hp", "engine_cylinders",
        "highway_mpg", "city_mpg", "popularity"]

X_train = df_train[base].fillna(0).values
w0, w = train_linear_regression(X_train, y_train)

y_pred = w0 + X_train.dot(w)

Filling with 0 effectively makes the model ignore that feature for that row (the term 0 × wj drops out). It's not always sensible (a car with 0 horsepower or 0 cylinders is unrealistic), but in practice 0 often works fine and keeps things simple. Filling with the mean is an alternative.

Compare predictions against the actual target with overlapping histograms:

sns.histplot(y_pred, color="red", alpha=0.5, bins=50)
sns.histplot(y_train, color="blue", alpha=0.5, bins=50)
The prediction distribution is shifted — even the peaks don't match, and the model often under-predicts. Eyeballing charts isn't enough; we need an objective metric. That's RMSE (root mean squared error), coming next.
9 · Evaluating With RMSE

📏 Root Mean Squared Error

To quantify how good a regression model is, we use RMSE. For each observation we take the difference between the prediction and the actual value, square it, average over all observations, and take the square root.

RMSE
RMSE = √( (1/m) · Σi (g(xi) − yi)2 )
g(xi) — the prediction for observation i.
yi — the actual value.
m — the number of observations. Squaring penalizes big errors; the root brings the metric back to the target's scale.

Worked example — predictions [10, 9, 11, 10.5] vs actuals [9, 9, 10.5, 11.5]: squared errors are [1, 0, 0.25, 2.25], mean = 0.875, so RMSE = √0.875 ≈ 0.93.

def rmse(y, y_pred):
    se = (y - y_pred) ** 2
    mse = se.mean()
    return np.sqrt(mse)
NumPy's .mean() handles the sum-and-divide in one call, so the whole metric is three lines. Lower RMSE = better model.
10 · Validating the Model

✅ Score on the Validation Set

The baseline mistake: we trained and scored on the training data. We should instead apply the model to the validation set and compute RMSE there. To avoid repeating the same prep code, wrap it in a prepare_X function that works for any dataframe.

def prepare_X(df):
    df_num = df[base]
    df_num = df_num.fillna(0)
    return df_num.values

X_train = prepare_X(df_train)
w0, w = train_linear_regression(X_train, y_train)

X_val = prepare_X(df_val)
y_pred = w0 + X_val.dot(w)
rmse(y_val, y_pred)            # validation RMSE
The pattern splits cleanly into a training part (touch only train) and a validation part (prepare val the same way → predict → score). This reusable function makes experimenting much faster.
11 · Feature Engineering

🛠️ Creating the Age Feature

Feature engineering is creating new features from existing ones. The year column is one of the most important price predictors — but the age of the car is more useful. The data was collected in 2017, so age = 2017 − year.

def prepare_X(df):
    df = df.copy()                     # don't mutate the caller's df
    features = base.copy()

    df["age"] = 2017 - df.year
    features.append("age")

    df_num = df[features].fillna(0)
    return df_num.values
Always df.copy() first — otherwise the function adds columns to the caller's dataframe (a side effect you don't want). Likewise copy base before appending, so the shared list isn't modified each call.

Adding age is a big win: validation RMSE drops from ~0.76 to ~0.51, and the predicted distribution now lines up far better with the actual prices.

12 · Categorical Variables

🔤 One-Hot Encoding

Categorical variables are categories — usually strings like make, model, fuel type, transmission. Note that number_of_doors looks numeric but is really categorical (2, 3, or 4 distinct door types).

We encode each category as a set of binary columns — one per value, set to 1 when the row matches, 0 otherwise. This is one-hot encoding.

for v in [2, 3, 4]:
    df["num_doors_%s" % v] = (df.number_of_doors == v).astype(int)

For high-cardinality columns like make, keep only the most common values via value_counts, and build a dictionary of the top categories to loop over inside prepare_X.

list(df.make.value_counts().head().index)   # top-5 makes

categorical = {
    "make": list(df.make.value_counts().head().index),
    "engine_fuel_type": [...],
    "transmission_type": [...],
    "driven_wheels": [...],
    "market_category": [...],
    "vehicle_size": [...],
    "vehicle_style": [...],
}

for c, values in categorical.items():
    for v in values:
        df["%s_%s" % (c, v)] = (df[c] == v).astype(int)
        features.append("%s_%s" % (c, v))

Adding number_of_doors barely helps; make improves RMSE by ~1%. But after throwing in all categoricals, something breaks — RMSE jumps to ~41 and the weights become enormous. We'll see why next.

One-hot encoding turns one categorical column into many binary features. It's the standard approach — covered in more depth in the classification session.
13 · Regularization

⚖️ Fixing Numerical Instability

The explosion comes from the normal equation's inverse. If two feature columns are duplicates (or near-duplicates, e.g. one is a linear combination of others), the Gram matrix XTX is singular and has no inverse.

Real data has noise, so the matrix is rarely exactly singular — NumPy still inverts it, but produces huge weights that wreck the model. The fix: add a small number to the diagonal of the Gram matrix before inverting. This keeps any column from being a duplicate of another and controls the weights.

Regularized Normal Equation
w = (XTX + r·I)-1 XT y
r — the regularization parameter: how much we add to the diagonal.
I — the identity matrix. Larger r → smaller, more controlled weights; r = 0 → plain linear regression. 
def train_linear_regression_reg(X, y, r=0.01):
    ones = np.ones(X.shape[0])
    X = np.column_stack([ones, X])

    XTX = X.T.dot(X)
    XTX = XTX + r * np.eye(XTX.shape[0])   # add r to the diagonal

    XTX_inv = np.linalg.inv(XTX)
    w_full = XTX_inv.dot(X.T).dot(y)
    return w_full[0], w_full[1:]
Regularization means "controlling" the weights so they don't grow too large. With it, the full-feature model finally beats every earlier version — but too much r degrades performance, so it must be tuned.
14 · Tuning the Regularization Parameter

🎚️ Finding the Best r

Use the validation set to pick r. Try a range of values, train with each, and compare validation RMSE (also watch the bias term — more regularization shrinks it).

for r in [0, 0.001, 0.01, 0.1, 1, 10]:
    w0, w = train_linear_regression_reg(X_train, y_train, r=r)
    y_pred = w0 + X_val.dot(w)
    print(r, w0, rmse(y_val, y_pred))
At r = 0 the bias and RMSE are huge; a little regularization fixes it. Around r = 0.01 the model performs well and hasn't started to degrade — a good, safe choice.
15 · The Final Model

🏁 Train on Full Data & Test

For the final model, combine train + validation into one full-train set, train once with the chosen r, then confirm on the untouched test set.

df_full_train = pd.concat([df_train, df_val]).reset_index(drop=True)
y_full_train = np.concatenate([y_train, y_val])

X_full_train = prepare_X(df_full_train)
w0, w = train_linear_regression_reg(X_full_train, y_full_train, r=0.01)

X_test = prepare_X(df_test)
y_pred = w0 + X_test.dot(w)
rmse(y_test, y_pred)          # ≈ validation RMSE → generalizes well
When test RMSE matches validation RMSE to ~3 decimals, the model generalizes well — the validation score wasn't just luck.

🚙 Predicting a Single Car

In production a car arrives as a dictionary (e.g. from a web form). Wrap it in a one-row dataframe, run prepare_X, predict, and undo the log with expm1.

car = {"make": "toyota", "year": 2015, ...}   # request from user

df_small = pd.DataFrame([car])
X_small = prepare_X(df_small)
y_pred = w0 + X_small.dot(w)
np.expm1(y_pred[0])           # predicted price in dollars
Take only the first element (one car → one prediction). The result is reasonably close to the true price — the model works end to end.
