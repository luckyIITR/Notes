1 · How This Round Works

🎙️ What They're Actually Testing

The interviewer gives you a problem like "implement linear regression from scratch" — meaning no scikit-learn for the algorithm itself. NumPy is allowed and expected.

They are watching for:

  • Do you understand the math behind the model (not just model.fit())?
  • Can you turn that math into clean code under mild pressure?
  • Can you explain your thinking out loud as you go?
Your game plan for every problem (say this out loud)
  1. Clarify: "Just to confirm — NumPy is fine, no sklearn for the model? Binary or multi-class? Should I include training, or just the predict step?"
  2. Explain the idea in one line before coding. ("Linear regression fits a line by minimizing squared error, and I'll use gradient descent.")
  3. Write a class with fit and predict — this structure impresses and is easy to extend.
  4. Test it on a tiny example at the end.
  5. Mention trade-offs: complexity, when it breaks, how you'd improve it.
Golden rule: narrate while you code. Silence looks like you're stuck even when you're not.

3 · Linear Regression ⭐

📈 Fitting a Line by Minimizing Error

The idea

Fit a straight line through the data so predictions are as close as possible to the real values. "Close" is measured by Mean Squared Error — the average of squared mistakes. We adjust the line's weights step by step to make that error smaller; this is gradient descent.

The steps
  1. Start with weights = 0.
  2. Predict y = X·w + b.
  3. Measure the error (prediction − actual).
  4. Nudge w and b in the direction that reduces error.
  5. Repeat many times.
The code
Pythonimport numpy as np

class LinearRegressionGD:
    def __init__(self, lr=0.01, n_iters=1000):
        self.lr = lr            # learning rate (step size)
        self.n_iters = n_iters
        self.w = None
        self.b = None

    def fit(self, X, y):
        n_samples, n_features = X.shape
        self.w = np.zeros(n_features)
        self.b = 0
        for _ in range(self.n_iters):
            y_pred = X @ self.w + self.b          # prediction
            error = y_pred - y
            dw = (1 / n_samples) * (X.T @ error)  # gradient w.r.t. weights
            db = (1 / n_samples) * np.sum(error)  # gradient w.r.t. bias
            self.w -= self.lr * dw                # update step
            self.b -= self.lr * db

    def predict(self, X):
        return X @ self.w + self.b
Bonus — the Normal Equation (closed-form)

No iterations. Good to mention; interviewers love it.

Pythondef linreg_normal_equation(X, y):
    X_b = np.c_[np.ones((X.shape[0], 1)), X]      # add bias column
    theta = np.linalg.inv(X_b.T @ X_b) @ X_b.T @ y
    return theta   # theta[0] = bias, theta[1:] = weights
Likely follow-up questions
"Why squared error and not absolute error?"
Squared error is smooth and easy to differentiate, and punishes big mistakes more.
"Gradient descent vs normal equation — when to use which?"
Normal equation is exact but slow for many features (it inverts a matrix, ~O(n³)). Gradient descent scales better to large/high-dimensional data.
"What does the learning rate do?"
Too high → it overshoots and never settles; too low → very slow training.
"What is the cost function?"
MSE = (1/n) Σ (y_pred − y)².
4 · Logistic Regression ⭐

🎯 Classification via the Sigmoid

The idea

Like linear regression, but for classification (yes/no). We pass the linear output through a sigmoid function that squashes any number into a probability between 0 and 1. Above 0.5 → class 1, below → class 0.

Key difference from linear: prediction goes through sigmoid, and the loss is log loss (cross-entropy) instead of MSE. Nicely, the gradient formula ends up looking almost identical.

The code
Pythonclass LogisticRegression:
    def __init__(self, lr=0.1, n_iters=1000):
        self.lr = lr
        self.n_iters = n_iters

    def _sigmoid(self, z):
        return 1 / (1 + np.exp(-z))

    def fit(self, X, y):
        n_samples, n_features = X.shape
        self.w = np.zeros(n_features)
        self.b = 0
        for _ in range(self.n_iters):
            linear = X @ self.w + self.b
            y_pred = self._sigmoid(linear)
            dw = (1 / n_samples) * (X.T @ (y_pred - y))
            db = (1 / n_samples) * np.sum(y_pred - y)
            self.w -= self.lr * dw
            self.b -= self.lr * db

    def predict_proba(self, X):
        return self._sigmoid(X @ self.w + self.b)

    def predict(self, X):
        return (self.predict_proba(X) >= 0.5).astype(int)
Likely follow-up questions
"Why not use MSE here?"
With sigmoid, MSE creates a bumpy (non-convex) surface that's hard to optimize. Log loss is convex, so gradient descent works cleanly.
"What is the sigmoid doing?"
Converting a raw score into a probability between 0 and 1.
"How would you handle more than 2 classes?"
Use softmax (multinomial logistic regression) or one-vs-rest.
"Is this a linear or non-linear model?"
The decision boundary is linear.
5 · K-Means Clustering ⭐

🎭 Assign-and-Update Clusters

The idea

Group unlabeled data into k clusters. Each cluster has a center (centroid). We repeat two steps until things stop moving: (1) assign each point to its nearest centroid, (2) move each centroid to the average of its points.

The steps
  1. Pick k random points as starting centroids.
  2. Assign: each point joins its nearest centroid.
  3. Update: recompute each centroid as the mean of its assigned points.
  4. Repeat until centroids stop changing.
The code (modular form)

fit reads top-down as the algorithm itself; each helper does one job. Keeping plain Python loops (no vectorization tricks) makes every step easy to explain out loud.

Pythonclass KMeans:
    def __init__(self, k=3, max_iters=100):
        self.k = k
        self.max_iters = max_iters

    # ── 1. random initialization ──────────────────────────
    def _init_centroids(self, X):
        m = X.shape[0]
        return X[np.random.randint(0, m, self.k)].copy()

    # ── 2. find the nearest centroid for one point ───────
    def _nearest_centroid(self, point):
        min_distance = float('inf')
        min_index = -1
        for c_idx in range(self.k):
            dist = np.linalg.norm(self.centroids[c_idx] - point)
            if dist < min_distance:
                min_distance = dist
                min_index = c_idx
        return min_index

    # ── 3. assign every point to its nearest centroid ────
    def _assign_clusters(self, X):
        clusters = [[] for _ in range(self.k)]
        for point in X:
            idx = self._nearest_centroid(point)
            clusters[idx].append(point)
        return clusters

    # ── 4. recompute each centroid as the mean of its points ──
    def _update_centroids(self, clusters):
        new_centroids = np.zeros_like(self.centroids)
        for i in range(self.k):
            if len(clusters[i]) > 0:
                new_centroids[i] = sum(clusters[i]) / len(clusters[i])
            else:
                new_centroids[i] = self.centroids[i]   # keep old if empty
        return new_centroids

    # ── 5. the high-level algorithm ──────────────────────
    def fit(self, X):
        self.centroids = self._init_centroids(X)
        for _ in range(self.max_iters):
            clusters = self._assign_clusters(X)
            new_centroids = self._update_centroids(clusters)
            if np.allclose(new_centroids, self.centroids):
                break                                  # converged
            self.centroids = new_centroids
        return self

    def predict(self, X):
        return np.array([self._nearest_centroid(p) for p in X])
Likely follow-up questions
"How do you choose k?"
The elbow method: plot total within-cluster distance for several k values and pick the "elbow" where it stops dropping fast. Or the silhouette score.
"Is K-Means guaranteed to find the best answer?"
No. It can get stuck in a local optimum, which is why we often run it several times with different starts (k-means++ helps pick smart starting points).
"It's unsupervised — what does that mean?"
There are no labels; it finds structure on its own.
"What if a cluster ends up empty?"
Reinitialize that centroid (the code keeps the old one to avoid a crash).
6 · K-Nearest Neighbors (KNN)

🧭 Vote of the Nearest Neighbors

The idea

To classify a new point, look at the k closest training points and take a majority vote. There is no real "training" — it just memorizes the data (called a lazy learner).

The code
Pythonclass KNN:
    def __init__(self, k=3):
        self.k = k

    def fit(self, X, y):
        self.X_train = X       # just store the data
        self.y_train = y

    def predict(self, X):
        preds = []
        for x in X:
            dists = np.linalg.norm(self.X_train - x, axis=1)
            k_idx = np.argsort(dists)[:self.k]      # k nearest indices
            k_labels = self.y_train[k_idx]
            preds.append(np.bincount(k_labels).argmax())  # majority vote
        return np.array(preds)
Likely follow-up questions
"What's the cost of prediction?"
Slow: O(n × d) per query because it compares to every training point. Fine for small data, bad for huge data.
"Why must you scale features first?"
Distance is dominated by large-range features. Always standardize.
"What if k is too small or too large?"
Too small → sensitive to noise; too large → over-smoothed and may miss real boundaries. Use an odd k for binary to avoid ties.
"Classification vs regression with KNN?"
Classification = majority vote; regression = average of neighbors' values.
7 · Decision Tree (Classification)

🌳 Recursive Yes/No Splits

The idea

Ask a series of yes/no questions about the features (e.g. "is feature 2 ≤ 0.5?") that best split the data into pure groups. "Pure" means mostly one class. We measure purity with Gini impurity (0 = perfectly pure).

The steps
  1. At each node, try every feature and threshold; pick the split that most reduces impurity.
  2. Split the data and repeat on each side (recursion).
  3. Stop when the node is pure, too small, or max depth is reached → make it a leaf that predicts the majority class.
The code
Pythonclass Node:
    def __init__(self, feature=None, threshold=None, left=None, right=None, value=None):
        self.feature = feature; self.threshold = threshold
        self.left = left; self.right = right
        self.value = value     # set only for leaf nodes

class DecisionTree:
    def __init__(self, max_depth=5, min_samples_split=2):
        self.max_depth = max_depth
        self.min_samples_split = min_samples_split

    def _gini(self, y):
        probs = np.bincount(y) / len(y)
        return 1 - np.sum(probs ** 2)

    def _best_split(self, X, y):
        best_gain, best_feat, best_thr = -1, None, None
        parent_gini = self._gini(y)
        n_samples, n_features = X.shape
        for feat in range(n_features):
            for thr in np.unique(X[:, feat]):
                left, right = y[X[:, feat] <= thr], y[X[:, feat] > thr]
                if len(left) == 0 or len(right) == 0:
                    continue
                child_gini = (len(left)/n_samples)*self._gini(left) + \
                             (len(right)/n_samples)*self._gini(right)
                gain = parent_gini - child_gini
                if gain > best_gain:
                    best_gain, best_feat, best_thr = gain, feat, thr
        return best_feat, best_thr

    def _build(self, X, y, depth):
        if depth >= self.max_depth or len(np.unique(y)) == 1 \
           or len(y) < self.min_samples_split:
            return Node(value=np.bincount(y).argmax())
        feat, thr = self._best_split(X, y)
        if feat is None:
            return Node(value=np.bincount(y).argmax())
        mask = X[:, feat] <= thr
        left = self._build(X[mask], y[mask], depth + 1)
        right = self._build(X[~mask], y[~mask], depth + 1)
        return Node(feature=feat, threshold=thr, left=left, right=right)

    def fit(self, X, y):
        self.root = self._build(X, y, 0)

    def _predict_one(self, x, node):
        if node.value is not None:
            return node.value
        branch = node.left if x[node.feature] <= node.threshold else node.right
        return self._predict_one(x, branch)

    def predict(self, X):
        return np.array([self._predict_one(x, self.root) for x in X])
Likely follow-up questions
"Gini vs Entropy?"
Both measure impurity; results are usually similar. Gini is slightly faster (no log).
"Why do trees overfit, and how do you prevent it?"
A deep tree can memorize the data. Limit it with max_depth, min_samples_split, or prune it.
"How does this become a Random Forest?"
Train many trees on random subsets of rows and features, then average/vote. This reduces overfitting.
"Do you need to scale features?"
No. Trees split on thresholds, so scale doesn't matter.
8 · Gaussian Naive Bayes

🧮 Bayes With Independent Features

The idea

Use probability (Bayes' theorem) to classify. For each class we ask: "how likely is this data point under this class?" We assume features are independent (that's the "naive" part) and bell-shaped (Gaussian). We pick the class with the highest probability. We work in log space to avoid tiny-number underflow.

The code
Pythonclass GaussianNB:
    def fit(self, X, y):
        self.classes = np.unique(y)
        self.mean, self.var, self.prior = {}, {}, {}
        for c in self.classes:
            Xc = X[y == c]
            self.mean[c] = Xc.mean(axis=0)
            self.var[c] = Xc.var(axis=0) + 1e-9     # avoid divide-by-zero
            self.prior[c] = len(Xc) / len(y)

    def _log_gaussian(self, x, mean, var):
        return -0.5 * np.sum(np.log(2*np.pi*var)) - 0.5*np.sum(((x-mean)**2)/var)

    def predict(self, X):
        preds = []
        for x in X:
            posteriors = [np.log(self.prior[c]) + self._log_gaussian(x, self.mean[c], self.var[c])
                          for c in self.classes]
            preds.append(self.classes[np.argmax(posteriors)])
        return np.array(preds)
Likely follow-up questions
"Why is it called 'naive'?"
Because it assumes all features are independent, which is rarely true — yet it still works well.
"Why work in log space?"
Multiplying many small probabilities underflows to zero; adding logs is stable.
"Where is it used a lot?"
Text/spam classification — fast and surprisingly strong.
9 · PCA — Dimensionality Reduction

📐 Project Onto Max-Variance Axes

The idea

Squeeze many features into a few while keeping as much information (variance) as possible. It finds new axes — principal components — that point in the directions where the data spreads out the most.

The steps
  1. Center the data (subtract the mean).
  2. Compute the covariance matrix.
  3. Find its eigenvectors and eigenvalues.
  4. Keep the eigenvectors with the largest eigenvalues (most variance).
  5. Project the data onto them.
The code
Pythonclass PCA:
    def __init__(self, n_components=2):
        self.n_components = n_components

    def fit(self, X):
        self.mean = X.mean(axis=0)
        Xc = X - self.mean
        cov = np.cov(Xc.T)
        eigenvalues, eigenvectors = np.linalg.eigh(cov)
        idx = np.argsort(eigenvalues)[::-1]            # largest variance first
        self.components = eigenvectors[:, idx[:self.n_components]]
        return self

    def transform(self, X):
        return (X - self.mean) @ self.components
Likely follow-up questions
"Why center the data first?"
PCA measures variance around the mean; without centering, the result is wrong.
"How do you pick the number of components?"
Keep enough to explain ~95% of total variance (look at the explained-variance ratio).
"Why must features be scaled?"
Otherwise a large-scale feature dominates the variance unfairly.
10 · Simple Neural Network (1 Hidden Layer)

🧠 Forward Pass + Backprop

The idea

Stack layers of weighted sums passed through an activation function (sigmoid). Forward pass makes a prediction; backpropagation sends the error backward to update every weight using the chain rule. This is just gradient descent on a bigger function.

The code
Pythonclass SimpleNN:
    def __init__(self, n_input, n_hidden, lr=0.1, n_iters=2000):
        rng = np.random.default_rng(42)
        self.W1 = rng.standard_normal((n_input, n_hidden)) * 0.1
        self.b1 = np.zeros(n_hidden)
        self.W2 = rng.standard_normal((n_hidden, 1)) * 0.1
        self.b2 = 0.0
        self.lr, self.n_iters = lr, n_iters

    def _sigmoid(self, z):
        return 1 / (1 + np.exp(-z))

    def fit(self, X, y):
        y = y.reshape(-1, 1)
        n = X.shape[0]
        for _ in range(self.n_iters):
            # ---- forward ----
            z1 = X @ self.W1 + self.b1
            a1 = self._sigmoid(z1)
            z2 = a1 @ self.W2 + self.b2
            a2 = self._sigmoid(z2)
            # ---- backward (chain rule) ----
            dz2 = (a2 - y) / n
            dW2 = a1.T @ dz2
            db2 = np.sum(dz2)
            dz1 = (dz2 @ self.W2.T) * a1 * (1 - a1)   # sigmoid derivative
            dW1 = X.T @ dz1
            db1 = np.sum(dz1, axis=0)
            # ---- update ----
            self.W2 -= self.lr * dW2; self.b2 -= self.lr * db2
            self.W1 -= self.lr * dW1; self.b1 -= self.lr * db1

    def predict(self, X):
        a1 = self._sigmoid(X @ self.W1 + self.b1)
        a2 = self._sigmoid(a1 @ self.W2 + self.b2)
        return (a2 >= 0.5).astype(int).ravel()
In a real interview you write the from-scratch version to prove you understand the math, then mention you'd reach for PyTorch in production. Below is the exact same network — one hidden layer, sigmoid output — built with the library.
The same thing with PyTorch
Pythonimport torch
import torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset

# ---- Data ----
X = torch.tensor([[35.], [40.], [45.], [60.], [65.], [70.]])
y = torch.tensor([[0.],  [0.],  [0.],  [1.],  [1.],  [1.]])

dataset = TensorDataset(X, y)
loader  = DataLoader(dataset, batch_size=2, shuffle=True)

# ---- Model ----
class StudentClassifier(nn.Module):
    def __init__(self):
        super().__init__()
        self.network = nn.Sequential(
            nn.Linear(1, 8),   # input -> hidden
            nn.ReLU(),
            nn.Linear(8, 1),   # hidden -> output
            nn.Sigmoid()       # probability in [0, 1]
        )

    def forward(self, x):
        return self.network(x)

# ---- Setup ----
model     = StudentClassifier()
criterion = nn.BCELoss()                                 # binary cross-entropy
optimizer = torch.optim.Adam(model.parameters(), lr=0.01)

# ---- Training ----
for epoch in range(100):
    model.train()
    total_loss = 0
    for batch_x, batch_y in loader:
        predictions = model(batch_x)
        loss = criterion(predictions, batch_y)

        optimizer.zero_grad()   # clear old gradients
        loss.backward()         # backprop (autograd does the chain rule)
        optimizer.step()        # update weights
        total_loss += loss.item()

    if epoch % 10 == 0:
        print(f"Epoch {epoch}  Loss: {total_loss:.4f}")

# ---- Inference ----
model.eval()
student = torch.tensor([[55.]])
with torch.no_grad():
    probability = model(student)
    prediction  = (probability > 0.5).float()

print("Probability:", probability.item())
print("Prediction:",  prediction.item())

Likely follow-up questions
"What is backpropagation?"
Using the chain rule to compute how much each weight contributed to the error, then nudging it the opposite way.
"Why initialize weights randomly, not to zero?"
If all weights are equal, every neuron learns the same thing ("symmetry"). Random breaks it.
"Why use ReLU instead of sigmoid in deep nets?"
Sigmoid gradients get tiny in deep networks (vanishing gradient); ReLU avoids that and is faster.
11 · Evaluation Metrics From Scratch ⭐

📊 Accuracy, F1, RMSE — Without Sklearn

Interviewers very often add: "now write the accuracy / F1 / RMSE yourself." Know these cold.

The code
Pythondef accuracy(y_true, y_pred):
    return np.mean(y_true == y_pred)

def confusion_matrix_binary(y_true, y_pred):
    tp = np.sum((y_true == 1) & (y_pred == 1))
    tn = np.sum((y_true == 0) & (y_pred == 0))
    fp = np.sum((y_true == 0) & (y_pred == 1))
    fn = np.sum((y_true == 1) & (y_pred == 0))
    return tp, tn, fp, fn

def precision_recall_f1(y_true, y_pred):
    tp, tn, fp, fn = confusion_matrix_binary(y_true, y_pred)
    precision = tp / (tp + fp) if (tp + fp) else 0
    recall    = tp / (tp + fn) if (tp + fn) else 0
    f1 = 2*precision*recall / (precision + recall) if (precision + recall) else 0
    return precision, recall, f1

# Regression
def rmse(y_true, y_pred):
    return np.sqrt(np.mean((y_true - y_pred) ** 2))

def r2_score(y_true, y_pred):
    ss_res = np.sum((y_true - y_pred) ** 2)
    ss_tot = np.sum((y_true - y_true.mean()) ** 2)
    return 1 - ss_res / ss_tot
Plain-English meanings
  • Precision — of everything I predicted positive, how much was actually positive. (Cost of false alarms.)
  • Recall — of all the real positives, how many did I catch. (Cost of misses.)
  • F1 — balance of precision and recall (harmonic mean).
Likely follow-up question
"When is accuracy a bad metric?"
On imbalanced data. If 99% of cases are "no", predicting "no" always gives 99% accuracy but is useless. Use precision/recall/F1 or AUC.
12 · Train/Test Split + Standardization

✂️ Small Helpers They Sometimes Ask For

Pythondef train_test_split(X, y, test_size=0.2, seed=42):
    rng = np.random.default_rng(seed)
    idx = rng.permutation(len(X))
    test_n = int(len(X) * test_size)
    test_idx, train_idx = idx[:test_n], idx[test_n:]
    return X[train_idx], X[test_idx], y[train_idx], y[test_idx]

def standardize(X_train, X_test):
    mean = X_train.mean(axis=0)
    std = X_train.std(axis=0) + 1e-9
    return (X_train - mean) / std, (X_test - mean) / std
Critical interview point: always compute mean/std on the training set only, then apply to the test set. Computing it on the full data leaks information from the test set — a classic mistake they look for.

14 · Common Conceptual Questions

💭 Asked Alongside The Code

Bias vs Variance
High bias = too simple, underfits. High variance = too complex, overfits. You want a balance.
Overfitting fixes
More data, simpler model, regularization (L1/L2), dropout, cross-validation, early stopping.
L1 vs L2 regularization
L1 (Lasso) can shrink weights to exactly zero (feature selection). L2 (Ridge) shrinks them smoothly toward zero.
Why split data?
To check the model works on data it has never seen, not just memorized.
Cross-validation
Split data into k folds, train on k−1, test on the last, rotate. Gives a more reliable score.
Supervised vs unsupervised
Supervised has labels (regression, classification); unsupervised doesn't (clustering, PCA).
Gradient descent types
Batch (all data per step), Stochastic (one sample), Mini-batch (small groups — most common).
15 · Final Tips for the Live Round

🎬 What Actually Wins The Round

  1. Always write a class with fit and predict. It's the expected, clean structure.
  2. Vectorize with NumPy (X @ w) instead of Python loops where you can — it shows maturity.
  3. Add tiny inline comments on the tricky lines (the gradient, the update step).
  4. Test on a small array at the end and say "let me sanity-check it."
  5. If you blank on the exact gradient, say so honestly and reason it out from the loss — interviewers value the thinking more than perfect recall.
  6. Talk through trade-offs at the end (complexity, when it fails). This often matters more than the code itself.
You already work with these models in practice — this round is just about showing the gears inside. Practice writing linear regression, logistic regression, and K-Means from a blank file 2–3 times each until they flow without looking. That covers the majority of what gets asked.
